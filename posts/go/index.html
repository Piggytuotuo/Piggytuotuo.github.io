Go语言基础

Posted Apr 8, 2024 Updated Apr 22, 2024
By Crystal Li
1 min read

Go Foundation

  1. 环境变量

    1
2

     GOROOT golang安装的路径   D:\programs\Go 不需要设置
 GOPATH 自己定义的工作空间，需要手动设置(在系统变量添加)

    Windows10设置GOPATH不生效的原因：

    1

     Go会在用户级环境变量中设置一个GOPATH，设置的全局变量被用户级的覆盖，删除用户级的即可。

    查看go相关环境变量：

    1

     go env

  2. go get下载库被墙Connection timeout

    1

     配置代理 go env -w GOPROXY=https://goproxy.cn

  3. go get下载的文件存储位置

    1
2
3
4
5

     go env 查看GO111MODULE变量（是否使用Go Module作为包管理工具）

 如果为空或off，则在GOPATH/pkg目录下

 如果为on，则在GOPATH/src目录下

  4. go.mod作用

    gomod为包管理工具，记录项目所有依赖包的仓库地址及版本号，每次构建编译根据地址和版本号拉取依赖包。

    生成go.mod文件：

    1

         go mod init 项目名

Go Package

package（项目文件夹）:包含多个xx.go

package main: 在main包中定义func main(){}，为程序的入口

package xxx: 在其他包中定义函数，在main包中通过import调用，一般来说package xxx即为当前目录文件夹name

特殊情况：

package main下的两个文件调用：

1
2
3

    package main
    --main.go
    --dup1.go

运行时：

1
2

    go run .
    go run main.go dup1.go
